Simple Linear Regression¶

Now the dataset is ready to be analyzed. I am going to first run a simple linear regression model. The dependent variable is PRICE and the independent random variable is CRIM, which represents the per capita crime rate. The package sklearn works as follows.

  • Create a LinearRegression object
  • Feed the data to it
  • View results

Before running regression, let's visualize the relationship between PRICE and CRIM.

The trend does not exhibit strong nonlinearity (you can try a test to be more quantitative). The homoscedasticity is also a reasonable assumption.

In [7]:
# Import the object from linear_model
from sklearn.linear_model import LinearRegression
# Create an object
lm = LinearRegression()
# Fit PRICE and CRIM
# Here is another trick here. np.array(bos['CRIM']) returns a 1D array. But the coefficient matrix provided to lm.fit has to be 2D. So we use reshape to make it 2D.
lm.fit(X=np.array(bos['CRIM']).reshape(-1,1), y=bos['PRICE'])
# Report result
print('Intercept: ',  lm.intercept_)
print('Coefficient: ', lm.coef_)

Intercept:  24.03310617412388
Coefficient:  [-0.41519028]

So the fitted linear model is PRICE = 24.03-0.42CRIM. The next step is to compute the p-value of the coefficient and its confidence intervals. However, sklearn does not provide that information. You can compte it following the formula taught in the class. Or you can use another package called statsmodels.

In [8]:
# Import statsmodels to do the linear regression
import statsmodels.api as sm
# We need the intercept, so add a constant column
X = sm.add_constant(np.array(bos['CRIM']))
y = bos['PRICE']
# Fit the model using OLS (ordinary least square, as opposed to generalized least square)
lm1 = sm.OLS(y,X).fit()
print(lm1.summary())

As you can see, the table summarizes the most important information compactly. For example

  • Number of observations: 506
  • $R^2$ is 0.151
  • The table of const and x1 summarizes the estimated coefficient/intercept (first column). The second column is the estimated standard error (so you can construct a confidence interval based on it). The third column is the t-statistic, and the fourth column is the $p$-value. Both variables (CRIM and constant) are significant (so reject $H_0$ that they are zero).
  • Durbin-Watson test tests the autocorrelation, i.e., whether the errors exhibit correlation. The $p$-value is 0.7, so we do not reject $H_0$ which states that the errors are i.i.d.

Regression Diagnostics¶

Next we conduct some basic diagnostics for the regression model. We first plot the residual as a function of the fitted value.

In [10]:
# The results object has an attribute resid, representing the residuals
residual = lm1.resid
fitted = lm1.fittedvalues
# Next we plot it
plt.scatter(fitted, residual)
plt.show()

The residuals do not seem to be i.i.d.; there is a downward trend in the fitted value. Next we check the QQ-plot (normal probability plot) for normality. Scipy has a function for that.

In [11]:
# We first create an axis handle to plot
fig=plt.figure()
ax = fig.add_subplot(111)
# Use the scipy package
st.probplot(residual, dist='norm', plot=ax)
plt.show()

The normality assumption doesn't seem to be satisfied either.

The table is similar to above. You can see the confidence interval and $p$-value for each variables. For example, AGE seems to be independent of the housing price. So let's test a model in which the variable AGE is absent.

In [13]:
print(lm2.f_test('AGE = 0'))

<F test: F=array([[0.00274601]]), p=0.9582293092054368, df_denom=492, df_num=1>

The $p$-value is exactly equal to the one produced by the table. In other words, we don't have to include AGE in the model. This f_test allows us to test whether multiple variables are zero.

In [14]:
